# 用户认证规范

- 使用 `get_user_model()` 或 `settings.AUTH_USER_MODEL`，禁止直接导入 Django 内置 `User`。
- 用户名、邮箱和手机号可用于登录；邮箱唯一，手机号有值时唯一。
- 页面统一展示 `user.display_name`，其顺序为昵称、脱敏手机号、用户名，不直接展示手机号注册产生的内部随机用户名。
- 个人中心可修改昵称、用户名和邮箱；手机号在验证码换绑能力完成前保持只读。
- 个人中心修改密码必须验证旧密码，成功后保持当前会话，不复用忘记密码流程。
- 头像保存至 `MEDIA_ROOT/avatars/<用户ID>/`，校验图片类型与大小，并在替换或删除用户时清理文件。
- 认证和资料修改页面必须使用 POST、CSRF，并对个人中心增加登录保护。
- 关闭第三方服务时，固定验证码流程不得访问 Redis、Celery 验证码任务或第三方 Provider。
