# 系统启用说明

## 环境准备

```bash
python -m pip install uv  # 已安装时跳过
uv sync --all-groups
cp .env.example .env
```

本地默认使用 SQLite。使用 MySQL 8 时在 `.env` 中设置 `DB_ENGINE=mysql` 及对应的 `DB_*` 变量。

## 本地启动

```bash
uv run python manage.py migrate
uv run python manage.py createsuperuser
uv run python manage.py runserver
```

- 主页：<http://127.0.0.1:8000/>
- 后台：<http://127.0.0.1:8000/admin/>

启动前检查项目：`make check`。

## 本地认证

- 登录支持用户名、邮箱或手机号。
- `AUTH_STYLE=picture` 使用图片，`AUTH_STYLE=video` 使用视频；各页面媒体路径通过 `AUTH_*_MEDIA` 配置。
- 本地不连接第三方服务，默认短信验证码为 `246810`，邮箱验证码为 `135790`。可在 `.env` 修改 `AUTH_FIXED_SMS_CODE` 和 `AUTH_FIXED_EMAIL_CODE`。
- `USE_THIRD_PARTY_SERVICES=False` 时验证码不写入 Redis、不投递 Celery 验证码或欢迎邮件任务，开发验证码 cache 固定使用进程内 `LocMemCache`。
- 本地固定验证码只用于开发，不得作为开放网络环境的安全方案。

## 个人中心与媒体

- 登录后点击导航栏头像或昵称进入个人中心，可修改头像、昵称、用户名、邮箱和登录密码。
- 修改密码必须输入正确旧密码；成功后当前设备保持登录。
- 头像写入 `media/avatars/<用户ID>/`。开发服务器在 `DEBUG=True` 时提供媒体访问，`media/` 不提交 Git。
