# 第十二批：本地验证码与消息时效

## 问题

- `base.py` 可能在 `dev.py` 覆盖第三方服务开关前绑定 Redis 验证码 cache。
- 固定验证码注册完成后仍投递欢迎邮件 Celery 任务，导致本地 Redis 出现任务数据。
- 退出登录成功消息会持续显示，且认证布局重复渲染消息区域。

## 完成内容

- 公共及开发配置默认使用 `LocMemCache`，仅生产开启第三方服务时绑定 Redis。
- 固定码模式的请求、校验、待发送码读取和 Celery 任务均在访问 cache 前返回。
- 本地注册不再投递欢迎邮件任务，确保固定码完整流程不访问 Redis broker。
- 统一使用共享消息组件，成功消息 3 秒后自动关闭，并提供手动关闭按钮。
- 增加淡出动画、空消息容器清理及无障碍状态提示。
